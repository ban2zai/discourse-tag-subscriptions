# discourse-tag-subscriptions

Discourse-плагин для удобного управления подписками на теги прямо со страницы настроек пользователя (`/u/:username/preferences/notifications`).

## Что делает

Добавляет секцию «Подписки на теги» в настройки уведомлений каждого пользователя. Пользователь видит сгруппированный список тегов и может одним кликом выбрать уровень уведомлений:

- **1 клик** — уведомление о новой теме с тегом (`watching_first_post`)
- **2 клика** — уведомление о каждом ответе в теме с тегом (`watching`)
- **3 клика** — отменить подписку

Изменения сохраняются вместе с нативной кнопкой «Сохранить изменения» Discourse — никаких отдельных кнопок.

## Возможности

- Группировка тегов по настроенным группам тегов Discourse
- Закреплённые теги вне групп (секция «Иные теги» вверху списка)
- Автовыбор родительского тега при выборе всех дочерних
- Алфавитная сортировка тегов внутри групп
- Порядок групп соответствует порядку в настройках плагина
- Цветовая индикация уровня подписки (два настраиваемых цвета)
- Кнопка «Скрыть» / «Показать» секцию (сохраняется в localStorage)
- Staff видит все группы тегов; обычные пользователи — только доступные им
- Поддержка кастомных display-имён тегов через CSS `::before` темы

## Требования

- Discourse (актуальная stable-версия)
- Теги и группы тегов должны быть включены в настройках форума

## Установка

В `app.yml` в секцию `hooks.after_code`:

```yaml
hooks:
  after_code:
    - exec:
        cd: $home/plugins
        cmd:
          - git clone https://github.com/ban2zai/discourse-tag-subscriptions.git
```

Затем пересборка контейнера:

```bash
./launcher rebuild app
```

## Настройка

После установки перейди в **Admin → Settings** и найди настройки с префиксом `tag_subscription`:

| Настройка | Описание | Пример |
|---|---|---|
| `tag_subscription_groups` | Имена групп тегов через `\|` в нужном порядке | `БГУ Разделы\|ЗГУ Разделы\|Техно Разделы` |
| `tag_subscription_pinned_tags` | Теги через `\|` для секции «Иные теги» вверху | `какой-раздел\|общее` |
| `tag_subscription_level1_color` | Цвет (hex) для уровня «новая тема» | `#e5b000` |
| `tag_subscription_level2_color` | Цвет (hex) для уровня «все ответы» | `#cc5500` |
| `tag_subscription_level1_label` | Подпись в легенде для уровня 1 | `Уведомление о новой теме с тегом` |
| `tag_subscription_level2_label` | Подпись в легенде для уровня 2 | `Уведомление о каждом ответе в теме с тегом` |
| `tag_subscription_hint` | Подсказка под легендой | `Нажмите 1 раз — только новые темы, 2 раза — все ответы, 3 раза — отменить` |

## Архитектура

Плагин состоит из двух частей:

**Ruby (`plugin.rb`)** — единственный эндпоинт `GET /tag-subscriptions/tag-groups.json`, доступный залогиненным пользователям. Возвращает группы тегов с учётом прав доступа пользователя и поля `parent_tag`. Staff получает все группы без фильтрации.

**JavaScript/Handlebars** — Glimmer-компонент в outlet `user-preferences-notifications`. Загружает группы через кастомный эндпоинт, текущие подписки через `/u/:username.json`. При сохранении делает `PUT /u/:username.json` только с тег-полями, не трогая остальные настройки пользователя.
